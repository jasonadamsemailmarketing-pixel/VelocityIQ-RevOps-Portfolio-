The AI module is inserted between Step 07 (Notify) and Step 08 (Task) in Scenario 1. It receives the fully enriched, scored, and graded lead data, calls the Gemini API, and injects both outputs into the downstream Slack card and HubSpot note.

Important: This adds only 2 new modules to Scenario 1 — one HTTP module (the API call) and one JSON parser (to extract the two outputs). The Slack and HubSpot Note modules are updated to inject the AI output, but not duplicated.

Step 1 — Get Your Free Gemini API Key

1

Go to Google AI Studio

Visit aistudio.google.com and sign in with your Google account. Click "Get API Key" in the left sidebar.

2

Create API Key

Click "Create API Key" → "Create API key in new project." Copy the generated key. Store it in Make.com → Data Stores or your environment variables — never hardcode it.

3

Verify the free tier limits

Gemini 1.5 Flash: 15 requests/minute and 1 million tokens/day on the free tier. At 300 leads/month, you'll use fewer than 10K tokens total per month. Comfortably free.

Step 2 — Configure the Make.com HTTP Module

Add an HTTP "Make a request" module immediately after the Route module (Step 06). Configure it as follows:

Module Configuration

URL: https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash-latest:generateContent Method: POST Headers: Content-Type: application/json x-goog-api-key: {{YOUR_GEMINI_API_KEY}} Body type: Raw (JSON)

Request Body (paste into Make.com body field)

{ "contents": [ { "parts": [ { "text": "You are a B2B sales intelligence assistant. Given the following inbound lead data, generate TWO outputs in valid JSON only. No preamble, no markdown backticks.\n\nLead Data:\n- Name: {{1.lead_name}}\n- Company: {{1.company_name}}\n- Job Title: {{1.job_title}}\n- Company Size: {{1.company_size}} employees\n- Industry: {{1.industry}}\n- ICP Grade: {{4.icp_grade}}\n- Lead Score: {{4.lead_score}}/80\n- Intent Timeline: {{1.timeline}}\n- Use Case: {{1.use_case}}\n- Assigned Rep: {{6.rep_name}}\n\nRespond with this exact JSON structure:\n{\n \"rep_briefing\": \"2-3 sentence briefing for {{6.rep_name}} explaining who this lead is, why they matter, and what to lead with on the call. Reference the grade and key signals.\",\n \"email_draft\": \"A concise 3-sentence first-touch email the rep can send immediately. Personalised to the lead's company, role and timeline. Professional but warm. Include a specific meeting ask.\"\n}" } ] } ], "generationConfig": { "temperature": 0.4, "maxOutputTokens": 400, "responseMimeType": "application/json" } }

Variable substitution: Replace {{1.lead_name}}, {{4.icp_grade}} etc. with the actual Make.com variable references from your previous modules. Module numbers depend on your scenario structure.

Step 3 — Variable Reference Table

Map these prompt placeholders to your Make.com module outputs:

PlaceholderSourceMake.com VariableNotes
{{lead_name}} Tally.so webhook Module 1 → "full_name" First + Last Name fields combined
{{company_name}} Tally.so or Enrichment Module 1 or 2 → "company" Prefer enriched value if available
{{job_title}} Tally.so webhook Module 1 → "job_title" Direct from form field
{{company_size}} Enrichment module Module 2 → "employee_count" Numeric value from enrichment
{{industry}} Enrichment module Module 2 → "industry" Fallback to form field if null
{{icp_grade}} Score module Module 4 → "icp_grade" Value: A, B, or C
{{lead_score}} Score module Module 4 → "lead_score" Numeric, 0–80
{{timeline}} Tally.so webhook Module 1 → "timeline" e.g. "Immediately", "1–3 months"
{{use_case}} Tally.so webhook Module 1 → "use_case" Free text field from form
{{rep_name}} Route module Module 6 → "assigned_rep_name" First name only for natural briefing

Step 4 — Parse the API Response

The Gemini response wraps the JSON output inside a content block. Add a JSON Parser module after the HTTP module to extract the two output fields.

Response Path to the JSON String

// The full response structure from Gemini looks like this: { "candidates": [{ "content": { "parts": [{ "text": "{\"rep_briefing\": \"...\", \"email_draft\": \"...\"}" }] } }] } // In Make.com, reference the text string as: {{HTTP_MODULE.data.candidates[].content.parts[].text}} // Pass this into a JSON Parser module, then access: {{JSON_PARSER.rep_briefing}} → Use in Slack card {{JSON_PARSER.email_draft}} → Use in HubSpot note

Pro tip: Add an error handler after the HTTP module. If the API call fails (rate limit, timeout), use Make.com's error directive to continue the scenario without the AI output — the Slack notification and task creation should never be blocked by a non-critical AI call.

Step 5 — Inject AI Output into Slack & HubSpot

Update the existing Slack (Step 07) and HubSpot Note modules to include the AI-generated content.

Testing Checklist

✓

Test the HTTP module in isolation

Run the module with hardcoded test values first. Confirm the API returns valid JSON with rep_briefing and email_draft keys before connecting to the rest of the scenario.

✓

Test the JSON parser with the actual response structure

Gemini wraps the JSON in candidates[0].content.parts[0].text. Confirm your parser path reaches the inner JSON string before marking done.

✓

Submit a test lead end-to-end

Submit the Tally form with test data and verify the Slack notification includes both the Rep Briefing and Email Draft sections below the standard lead fields.

✓

Test error handling

Temporarily break the API URL and confirm the scenario continues to execute (Slack notification still fires, task still creates) even when the AI module fails. The AI output is enrichment — never a dependency.
